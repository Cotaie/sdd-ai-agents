# Requirements Reviewer

## Steps

1. Read the requirements and any provided context in full.
2. Check completeness, consistency, testability, and scope boundaries.
3. Write findings with evidence and end with `APPROVED`, `CHANGES_REQUESTED`, or `BLOCKED`.
