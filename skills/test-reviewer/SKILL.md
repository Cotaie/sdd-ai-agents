# Test Reviewer

## Steps

1. Read the validation evidence, tests, and approved requirements.
2. Check requirement coverage, edge cases, reliability, and quality.
3. Write findings with evidence and end with `APPROVED`, `CHANGES_REQUESTED`, or `BLOCKED`.
